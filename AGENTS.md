# Repository Guidelines

Declarative Nix flake for personal machines, homelab, and lab cloud infra.

## Quick Commands

- `nix develop`: enter dev shell.
- `nix fmt`: run project formatters.
- `nix flake check`: run CI-equivalent checks.
- `nix build .#nixosConfigurations.<host>.config.system.build.toplevel`: build a NixOS host.
- `nix build .#darwinConfigurations.<host>.system`: build a nix-darwin host.
- `just nix`: switch the current machine via `nh`.
- `just lab`: apply `infra/lab` with OpenTofu and SOPS-injected secrets.
- `just update`: update `flake.lock` and in-tree package pins.

## Project Structure

- `flake.nix` / `flake.lock`: flake entry and lockfile (`flake.nix` is generated).
- `modules/`: auto-imported flake-parts modules.
- `modules/flake/`: inputs, overlays, devshell, formatting, host builders, template export.
- `modules/nixos/`, `modules/home/`, `modules/darwin/`: reusable modules.
- `modules/hosts/<hostname>.nix`: host-specific composition.
- `pkgs/by-name/<name>/package.nix`: in-tree packages (`pkgs/default.nix` provides overlay/patches).
- `configs/`: shared configs and encrypted secrets.
- `infra/lab/`: OpenTofu code.
- `templates/`: flake templates exported from `templates/default.nix`.

## Conventions

- Put reusable logic in `modules/nixos/**`, `modules/home/**`, and `modules/darwin/**`.
- Keep host-specific changes in `modules/hosts/<hostname>.nix`.
- Keep package updates in `pkgs/by-name/<name>/package.nix`.
- Formatting uses treefmt tools: `nixfmt`, `stylua` (2-space indent), `prettier`, and OpenTofu formatter support.

## Validation

- Run `nix fmt` and `nix flake check` before pushing.
- Build each affected host output for config changes.
- For infra changes, run `just lab` or at minimum `tofu plan` via `sops exec-env`.

## Dendritic Flake Rules

- The flake is generated by flake-file dendritic wiring (`modules/flake/dendritic.nix`), with modules discovered via import-tree.
- Do not hand-edit `flake.nix`; change `modules/**/*.nix` and regenerate with `nix run .#write-flake`.
- Add new flake behavior in focused files like `modules/flake/<topic>.nix`.
- Modules extend outputs via aspect keys such as `flake.modules.<class>.<aspect>` and `perSystem`.
- If generated files are stale, `nix flake check` will fail.
